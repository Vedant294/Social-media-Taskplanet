Project Overview

What is TaskPlanet Social?

TaskPlanet Social is a production-grade full-stack social posting web application built for the 3W Full Stack Internship Assignment. Inspired by the TaskPlanet app's social feed, it allows users to create accounts, share posts with text and images, interact through likes and comments, and explore a live public feed — all in real time.

The application is fully deployed, mobile-responsive, and built with a clean modern UI featuring glassmorphism design, smooth animations, skeleton loading, and optimistic UI updates for an instant feel.

Key Features

Everything That Was Built

🔐
JWT Authentication
Secure signup & login with bcryptjs password hashing (12 salt rounds). Tokens stored in localStorage with expiry validation on mount.
📝
Create Posts
Post text, images, or both. Images uploaded to Cloudinary CDN with auto-optimization. Neither field is mandatory — either one is enough.
🌍
Public Feed
Infinite scroll feed showing all posts from all users. Skeleton loading cards for perceived performance. Stagger animation on load.
❤️
Like / Unlike
Toggle likes with instant optimistic UI update. Count updates immediately before API response. Pop animation on like. Stores usernames of likers.
💬
Comments
Full comment modal with all comments listed. Optimistic append on submit. Live count update. Stores username, name, and avatar of commenter.
🔍
Search & Filter
Debounced search (300ms) by post content or username. Filter tabs: All Posts, Most Liked, Most Commented — all server-side sorted.
🔥
Trending Sidebar
Trending hashtags panel with click-to-search. Who to Follow suggestions panel. Visible on desktop, hidden on mobile for clean layout.
📱
Fully Responsive
Mobile-first design. Bottom-sheet comment modal on mobile. Touch-friendly 44px tap targets. Tested at 375px, 768px, and 1200px.

Backend

API Endpoints

Method Route Auth Description
POST /api/auth/signup Public Register new user, returns JWT
POST /api/auth/login Public Login, returns JWT + user object
GET /api/posts Public Paginated feed with filter & search
POST /api/posts JWT Create post (text + image upload)
PUT /api/posts/:id/like JWT Toggle like — adds or removes
POST /api/posts/:id/comment JWT Add comment to post
DELETE /api/posts/:id JWT Delete own post (author only)
Database

MongoDB Schema (2 Collections)

Collection 1 — users

Stores name, username (unique), email (unique), hashed password, avatar URL, bio. Password uses select: false — never returned in API responses. Timestamps auto-added.

Collection 2 — posts

Stores denormalized author info (no JOIN needed on feed load), post text, Cloudinary image URL, embedded likes array (userId + username), and embedded comments array (full comment object with timestamps). Indexed on createdAt for fast feed sorting.

Security

Security Measures

  • ✅ Passwords hashed with bcryptjs — 12 salt rounds, never stored plain
  • ✅ JWT HS256 tokens — 7-day expiry, validated on every protected request
  • ✅ password field has select: false — never accidentally returned
  • ✅ Author-only delete — server checks post.author.userId === req.user._id
  • ✅ Rate limiting — 100 requests per 15 minutes per IP
  • ✅ CORS — only whitelisted origins allowed
  • ✅ File upload — images only (mimetype check), max 5MB enforced by Multer
  • ✅ Generic auth errors — doesn't reveal which field is wrong
  • ✅ All secrets in .env — gitignored, .env.example committed
